# ✅ 문제 구성

## 1️⃣ 재귀 기반 이진 탐색 구현

### ▪ 문제 설명

정렬된 정수 배열에서 특정 값을 **재귀 함수**를 이용해 탐색하라.

### ▪ 구현 파일

```
recursive_binary_search.c
```

### ▪ 핵심 개념

- 분할 정복 (Divide & Conquer)
- 재귀 호출 구조
- 시간 복잡도: **O(log n)**

### ▪ 동작 원리

1. 중앙값(mid)을 계산
2. 찾는 값이 mid보다 작으면 왼쪽 탐색
3. 크면 오른쪽 탐색
4. 같으면 탐색 성공

---

## 2️⃣ 순차 탐색 vs 이진 탐색 성능 비교

### ▪ 문제 설명

정렬된 배열에서 특정 값을 찾을 때:

- 순차 탐색 (Linear Search)
- 이진 탐색 (Binary Search)

두 알고리즘의 **비교 횟수**를 출력하라.

### ▪ 구현 파일

```
linear_vs_binary_search.c
```

### ▪ 실험 조건

- 배열 크기: 100
- 데이터: 1 ~ 100
- 찾는 값: 95 (배열 후반부 값)

### ▪ 실행 예시

```
찾는 값: 95

순차 탐색 성공 (인덱스 94)
순차 탐색 비교 횟수: 95

이진 탐색 성공 (인덱스 94)
이진 탐색 비교 횟수: 7
```

### ▪ 시간 복잡도 비교

| 알고리즘 | 시간 복잡도 |
|----------|------------|
| 순차 탐색 | O(n) |
| 이진 탐색 | O(log n) |

### ▪ 학습 포인트

- 이론적인 시간복잡도 차이를 실제 실행으로 확인
- 데이터 크기가 커질수록 성능 차이 극대화

---

## 3️⃣ 해시 테이블 (선형 조사법)

### ▪ 문제 설명

정수 데이터를 저장하는 해시 테이블을 구현하고,  
충돌 발생 시 **선형 조사법(Linear Probing)** 으로 처리하라.

### ▪ 구현 파일

```
hash_linear_probing.c
```

### ▪ 핵심 개념

- 해시 함수: `key % TABLE_SIZE`
- 충돌 처리: 다음 인덱스로 이동
- 시간 복잡도 (평균): **O(1)**

### ▪ 특징

- 빠른 탐색 가능
- 테이블이 가득 차면 성능 저하
- 클러스터링 문제 발생 가능

---

## 4️⃣ 문자열 해시 테이블 구현

### ▪ 문제 설명

문자열을 저장하는 해시 테이블을 구현하라.

### ▪ 구현 파일

```
hash_string.c
```

### ▪ 해시 함수 방식

- 문자열의 아스키 코드 합을 이용
- `sum % TABLE_SIZE`

### ▪ 학습 포인트

- 문자열 해싱 원리 이해
- strcmp / strcpy 활용
- 충돌 처리 방식 동일 (선형 조사법)

# 💡 핵심 정리

- 이진 탐색은 반드시 **정렬된 배열**에서만 사용 가능하다.
- 데이터 크기가 커질수록 O(log n)의 위력이 커진다.
- 해싱은 평균적으로 가장 빠른 탐색 방법이다.
- 하지만 해시는 충돌 처리 전략이 매우 중요하다.
